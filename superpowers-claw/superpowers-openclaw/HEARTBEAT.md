# HEARTBEAT.md — Health Check

> Quick check-in to verify I'm on track.

---

## What Is a Heartbeat

Quick check-in to verify:
1. Working on the right thing
2. Approach is working
3. No blockers
4. Process being followed

---

## When

**Auto:** After each major phase, after each batch, when starting new task
**Manual:** "Heartbeat check" or "How's it going?"

---

## What I Show

```
## Heartbeat

Phase: [Design/Plan/Execute/Verify]
Completed: [x] Task 1, [x] Task 2
In Progress: Task 3
Next: Complete Task 3, run verification
Blockers: [None / Describe]
```

---

## What I Ask

1. Is this still what you want?
2. Is this approach working?
3. Any concerns?
4. Want to adjust anything?

---

## Your Control

- Trigger anytime
- Skip if not needed
- Change frequency
- Override any process
